# Operator APIs (aclnn)

##  Instructions

To make it easier to call operators, a set of C-based APIs (with the **aclnn** prefix) are provided. You do not need to provide the Intermediate Representation (IR) definition to facilitate efficient model and application development. This method is called single-operator API calling, or aclnn calling for short.

When calling operator APIs, you need to reference the dependency header file and library file. Generally, the header file is stored in `${INSTALL_DIR}/include/aclnnop` and the library file is stored in `${INSTALL_DIR}/lib64`.

- Header file: 1. (Recommended) Reference the general operator header file **aclnn\_ops\_***\$\{ops\_project\}***.h**. 2. Reference a single-operator API header file **aclnn\_***\****.h** as needed.
- Library file: Reference the general operator library file **libopapi\_***\$\{ops\_project\}***.so** as needed.

*\$\{INSTALL\_DIR}* indicates the file path after CANN is installed. *\$\{ops\_project}* indicates an operator repository (such as math, nn, cv, and transformer). Set this parameter to the actual operator repository name.

## APIs

> **Deterministic Computing:**
>
> - Configuration: Due to different CANN or NPU models, multiple runs of the same operator may have different results. Under the same conditions (platform, device, version, and other random parameters), deterministic computing can be enabled for some operator APIs by using `aclrtCtxSetSysParamOpt` (see [AscendCL API (C)](https://www.hiascend.com/document/detail/en/canncommercial/850/API/appdevgapi/aclcppdevg_03_0004.html)). This ensures that the results of multiple runs are the same.
> - Performance: An operator using deterministic computing is usually slower than using non-deterministic computing. Therefore, the performance of a single model run may deteriorate. However, in scenarios such as experiments and debugging, where the same results must be obtained upon multiple runs for fault locating, deterministic computing can improve efficiency.
> - Threads: The deterministic status can be set only once in the same thread. If the deterministic status is set multiple times, the last valid setting is used. A valid setting refers to a deterministic status that has been used to deliver an operator task. If no operator task is delivered, only the deterministic variable is enabled but it is not delivered to the operator. As a result, the operator is not executed.
>   Solution: You are not advised to set the deterministic property multiple times for a thread. This problem exists no matter whether the binary function is enabled or not. It will be solved in later versions.
